Close three corner-case contract violations surfaced by the Phase 3 code review (03-REVIEW.md CR-01/02/03) and verified as gaps in 03-VERIFICATION.md. The happy-path goal for Phase 3 is observably achieved (258 tests pass; 5/6 SC fully verified; SC-5 partial), but the wire-protocol and exit-code contracts have three sharp edges:

1. CR-01 — Torn RunStart/RunEnd framing on reader/cache errors. `engine::run_one` emits `RunStart` at step 3 (line 208) then propagates `MinerError::Scan` via `?` from `GapDetector::detect` (line 250), `cache.get_or_build` (line 313), and the `ScanError::Io` / `ScanError::Miner` match arms (lines 370-373) WITHOUT emitting a closing `RunEnd`. Consumers see an orphaned `RunStart` envelope — a D-09 / OUT-04 framing invariant violation.

2. CR-02 — SIGINT exit-code routing bypassed on non-preflight errors. `main.rs:107-111` uses `?` to short-circuit `compute_exit_code` on the Err arm of `handle_scan_subcommand`. anyhow's `Termination` prints + exits 1, ignoring the cancel flag. D3-24 mandates `cancelled → 130 regardless of tier`. SIGINT + reader/IO error currently yields exit 1 instead of 130. The fix maps the catch-all Err arm to `RunOutcome::PreflightFailed` (NOT `HadScanErrors`) so that under cancel=false the historical exit-1 semantics are preserved for Phase 6 MCP/HTTP wrappers per D3-24, while under cancel=true the cancel-flag short-circuit inside `compute_exit_code` wins → 130.

3. CR-03 — String-match dispatch on `MinerError::Scan`. `main.rs:293-299` demotes `MinerError::Scan(msg)` to `PreflightFailed` via `msg.starts_with("unknown scan:")`. The matching string is produced in `engine/mod.rs:194` via `format!`. A future refactor of the format string silently reclassifies unknown-scan preflight failures as runtime errors with no compile-time gate. The typed `PreflightCode::UnknownScan` path through `engine::preflight::resolve_scan` is bypassed by run_one's inlined `registry.get`.

Purpose: Restore the D-09 framing invariant (every run terminates with RunEnd), restore the D3-24 cancel-overrides-everything exit-code rule WHILE preserving the exit-1 semantics for non-preflight engine catch-all errors, and eliminate the fragile string-match dispatch by routing the engine through the typed `engine::preflight::resolve_scan` helper that already returns `WireError(PreflightCode::UnknownScan)`. After Plan 03-07 lands, the verifier can re-run and mark all six ROADMAP success criteria fully verified.
