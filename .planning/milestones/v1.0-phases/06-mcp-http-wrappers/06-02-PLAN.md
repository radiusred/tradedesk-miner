    3. `## The seven Finding variants` — Bulleted list naming every variant from `crates/miner-core/src/findings/mod.rs` line 545's `Finding` enum, with a 1-2 line description of when each is emitted:
       - `RunStart` — first line of every run; carries `run_id`, `started_at_utc`, `code_revision`, `schema_version`.
       - `Result` — the main payload; one per (scan x instrument(s) x timeframe x window x param-point).
       - `ScanError` — mid-run failures (preflight pass but Scan::run errored or the kernel rejected the inputs).
       - `GapAborted` — strict gap policy fired and aborted before any `Result` was emitted; carries the full gap manifest.
       - `DryRun` — emitted only under `--dry-run`; carries the resolved job graph + `planned_job_count`.
       - `SweepSummary` — last line of a `miner sweep` invocation; carries `SweepTotals` + per-`scan_id` `FdrFamilySummary` entries.
       - `RunEnd` — final line of every run (after `SweepSummary` if a sweep); carries `ended_at_utc` + `RunSummary` + `PerScanCounts`.
    4. `## Common envelope fields` — Per the FOUND-03 / D-21 / D-22 lock, every `Result` variant carries these:
       - `schema_version` (u32; locked at 1; bump signals a non-additive break)
       - `scan_id_at_version` (e.g. `"stats.autocorr.ljung_box@1"`) — the scan + version pair
       - `run_id` (RFC 4122 UUID, copied from the parent `RunStart`)
       - `param_hash` (blake3 hex over the canonical sorted-key param JSON; identical params -> identical hash regardless of object key order)
       - `code_revision` (string; `git describe --always` at build time, injected by `build.rs`)
       - `data_slice` (see next section)
       - `instruments` (`Vec<InstrumentSpec>`; one element for Single-arity scans, two for Pair-arity / CROSS)
       - `timeframe` (string; e.g. `"15m"`, `"1h"`, `"1d"`)
       - `params` (the resolved param map; defaults filled in)
       - `effect` (see "Effect block" section)
       - `raw` (BTreeMap of `RawArray` entries; base64-encoded with dtype + shape; see "Raw arrays" section)
       - `repro` (Option — only populated when caller opted into bootstrap / null hygiene; see "Reproducibility envelope" section)
       - `dsr` (Option; reserved for v2 Deflated Sharpe Ratio; always `null` in v1)
       - `fdr_q` (Option; populated only when a `SweepSummary` carries this finding in a family — null otherwise)
    5. `## data_slice` — Sub-block. Per crates/miner-core/src/findings/mod.rs lines 63-117:
       - `range: TimeRange { start_utc, end_utc }` — the actual continuous range used by the scan (post gap-partitioning under `continuous_only` policy).
       - `sources: Vec<Source>` — one entry per `(source_id, symbol, side, timeframe)` quartet contributing to the finding (D4-03).
       - `gap_manifest_ref: Option<...>` — pointer to a separately-emitted gap manifest if the run produced one.
       - `gap_manifest: Option<GapManifest>` — full inline gap manifest under `continuous_only`; tuples of `(start, end, reason)` for missing daily files / corrupt files / intra-day holes against the trading calendar.
       - Two-leg / CROSS semantics: under Pair-arity scans, `gap_manifest` is the INTERSECTION of both legs' gaps (D4-04). The continuous sub-range emitted is the maximal gap-free intersection.
    6. `## Effect block` — Per crates/miner-core/src/findings/mod.rs lines 174-203:
       - `metric: String` — short identifier of the test statistic (e.g. `"autocorr_lb_q"`, `"vr_minus_one"`).
       - `value: f64` — the canonical effect statistic.
       - `p_value: Option<f64>` — present when the scan emits a p-value (most do; some emit only a stat).
       - `n: Option<u64>` — sample size used.
       - `ci95: Option<[f64; 2]>` — populated only when the caller opted into block / stationary bootstrap (HYG-03).
       - `effect_size: Option<EffectSize>` — populated post-HYG-01; `EffectSize { kind, value, ... }` where `kind` is `cohens_d` / `hedges_g` / `cliffs_delta` / `vr_minus_one` etc. per D5-03.
       - `extra: BTreeMap<String, serde_json::Value>` — per-scan extras (e.g. `acf_array`, `dof`, `lags` for Ljung-Box; `lag_grid`, `argmax_lag` for lead-lag CCF). See `docs/scan_catalogue.md` for the full per-scan key list.
    7. `## Raw arrays` — base64-with-shape pattern. `Raw { /* BTreeMap<String, RawArray> */ }` -> `RawArray { dtype, shape, b64 }`. `dtype` is one of `"<f8"`, `"<f4"`, `"<i8"`, `"<i4"`, `"<u8"`, `"<u4"`. `shape` is a JSON array of unsigned ints. `b64` is the standard-base64-encoded raw little-endian bytes (no padding tricks). Show the canonical decode one-liner: `np.frombuffer(base64.b64decode(s), dtype="<f8").reshape(shape)`. Cross-link to `docs/examples/decode_finding.py`.
    8. `## Reproducibility envelope` — per D5-05 / `ReproEnvelope`:
       - `master_seed: u64` — the seed user passed via `--seed` or the sweep manifest `[sweep].seed`.
       - `job_seed: u64` — derived deterministically via `derive_job_seed(master_seed, scan_id, instrument_spec, timeframe, window, params)` using blake3. Equal seeds across re-runs produce byte-identical findings.
       - `bootstrap: Option<BootstrapSpec>` — `BootstrapSpec { method, n_iter, block_length, seed }`. `method` is `"stationary"` (Politis-Romano) or `"block"`.
       - `null: Option<NullSpec>` — `NullSpec { method, n_iter, seed }`. `method` is `"circular_shift"` or `"iaaft"`.
