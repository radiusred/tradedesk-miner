    - `#[inline] pub(super) fn lead_lag_ccf(a: &[f64], b: &[f64], max_lag: usize) -> LeadLagResult` — implementation:
      1. Compute mu_a = mean(a), mu_b = mean(b), sigma_a = std(a, ddof=0), sigma_b = std(b, ddof=0)
      2. For each lag in -max_lag..=max_lag (sequential): compute Pearson correlation between a and b shifted by `lag`. Specifically:
         - lag > 0 means a_t correlates with b_{t-lag} (a leads b by lag): cor = (1/N_eff) * Σ_{t=lag..n} (a_t - mu_a)(b_{t-lag} - mu_b) / (sigma_a * sigma_b), where N_eff = n - |lag|
         - lag < 0 means b leads a by |lag|: cor = (1/N_eff) * Σ_{t=|lag|..n} (a_{t-|lag|} - mu_a)(b_t - mu_b) / (sigma_a * sigma_b)
         - lag == 0: full-sample Pearson
      3. argmax_lag = lags[ccf_values.iter().position_max_by(|x| x.abs())]  — absolute argmax (the strongest lead/lag signal, regardless of sign)
    - Sequential lag loop (NOT par_iter — determinism); cancel-poll between lag iterations
