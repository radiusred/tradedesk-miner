<objective>
Phase 5 Plan 05 is the FINAL plan — wires the CLI surface (`miner sweep`,
universal `miner scan` hygiene flags), publishes the sweep-manifest JSON schema
via `cargo xtask gen-schema`, and closes Phase 5 with integration tests
and README documentation.

CLI surface added:

1. **`miner sweep <manifest.toml>` subcommand:**
   - Positional `manifest: PathBuf`.
   - `--dry-run` — short-circuits to `DryRunFinding` emission per Plan 05-04.
   - `--seed N` — overrides `[sweep].seed` in the manifest.
   - `--bootstrap stationary|block` — overrides `[hygiene].bootstrap`.
   - `--bootstrap-n N` — overrides `[hygiene].bootstrap_n`.
   - `--null phase_scramble|circular_shift` — overrides `[hygiene].null`.
   - `--null-n N` — overrides `[hygiene].null_n`.
   - Exit code routing: 0 = clean, 1 = preflight rejection, 2 = mid-stream ScanError, 130 = SIGINT.

2. **`miner scan` universal hygiene flags** (extending Plan 03-05's ScanArgs):
   - Same five flags as sweep: `--bootstrap`, `--bootstrap-n`, `--null`, `--null-n`, `--seed`.
   - `ScanArgs::to_scan_request` populates the new `ScanRequest` fields (from Plan 05-03) — `master_seed`, `bootstrap_method`, `bootstrap_n`, `null_method`, `null_n`. (Sweep populates `job_seed` per-job; single-shot CLI leaves `job_seed = None` and engine derives it on the fly when bootstrap/null requested.)

3. **`cargo xtask gen-schema` extension:**
   - Existing call regenerates `schemas/findings-v1.schema.json`.
   - NEW call regenerates `schemas/sweep-manifest-v1.schema.json` via `schemars::schema_for!(SweepManifest)`. Requires `SweepManifest`/`SweepConfig`/`HygieneBlock`/`FdrConfig`/`JobBlock` to derive `JsonSchema` (add the derive if not already present — Plan 05-04 may have included it).
   - Both schemas regenerate deterministically; `git diff schemas/` after a fresh regen is empty.

4. **Integration tests** (`crates/miner-cli/tests/`):
   - `sweep_subcommand_smoke.rs` — spawn the built `miner` binary; `miner sweep manifest.toml` against a synthetic cache produces the expected envelope sequence; exit 0.
   - `sigint_mid_sweep.rs` — spawn `miner sweep` with the `--sleep-after-first-finding-ms 5000` test-internal flag; deliver SIGINT; assert exit 130 + streamed Results persist + NO SweepSummary written. Mirrors `sigint_preserves_stream.rs` from Plan 03-06.

5. **Documentation:**
   - `README.md` Quickstart section gains a `miner sweep example.toml` example with a minimal manifest showing scan + 2 instruments + 1 timeframe + 1 window + 1 param array + `[hygiene]` block. The Quant-agent's typical workflow.
   - `tests/REFERENCE-VERSIONS.md` extends with R 4.x + `tseries`/`stats` pinning (the source-of-truth for BH-FDR + Politis-White block-length goldens). Plan 05-02 may have introduced unmasked goldens; this plan finalises the provenance documentation.

Purpose: This is the contract-closing plan. After Plan 05-05, the user can:
1. Author a TOML manifest.
2. `miner sweep example.toml` — streams JSONL findings with effect sizes, bootstrap CIs, null p-values, BH-FDR-adjusted q-values, and a reproducible seed in every `repro` envelope.
3. `miner scan stats.autocorr.ljung_box@1 --bootstrap stationary --bootstrap-n 1000 --seed 0xDEAD ...` — single-shot with the same hygiene contract.
4. `miner sweep example.toml --dry-run` — see the planned job graph + count.
5. SIGINT a long-running sweep — keep every already-streamed finding.

Output: CLI subcommand + universal flags + JSON schema export + integration
tests + README Quickstart; Phase 5 sign-off ready.
