<objective>
Phase 5 Plan 02 implements the FIVE hand-rolled statistical kernels in a new
`crates/miner-core/src/scan/hygiene/` module. Pure math; no I/O; no engine
wiring; no serde dependencies inside the kernel signatures. Each kernel is a
public function on `&[f64]` + scalar parameters + a `seed: u64`, mirroring the
Phase 4 `scan/anom/adf/kernel.rs` + `scan/ljung_box/kernel.rs` discipline that
shipped 22 scans already.

The five kernels:

1. **effect_size.rs** — `cohens_d`, `hedges_g`, `cliffs_delta`, `vr_minus_one`.
   Each <30 LOC; hand-computed test vectors plus degenerate-input edge cases
   (n=0, n=1, equal groups → NaN).

2. **bootstrap.rs** — `stationary_bootstrap_ci` + `block_bootstrap_ci`, both
   generic over a stat closure `F: Fn(&[f64]) -> f64`. Uses
   `Xoshiro256PlusPlus::seed_from_u64(seed)` (NEVER `SmallRng` — non-portable
   per the upstream Rand Book). Inner resample loop is SEQUENTIAL (rayon-only
   at the per-job outer loop, never inside a single bootstrap call — Plan 04-05
   ADF AIC-lag precedent). Plus `block_length_pwppw(values)` — the
   Politis-White (2004) + Patton-Politis-White (2009) automatic block-length
   selector, floored at `max(3, ceil(n^(1/3)))`.

3. **null.rs** — `circular_shift_null_p` (trivial ~15-LOC rotation of `Vec<f64>`
   by uniform offset in `[1, n-1]`; offset 0 rejected because it is the identity
   transform). IAAFT phase-scramble (Theiler 1992) is INCLUDED if and only if
   `realfft = "3"` is also added to the workspace this plan (the executor
   decides — see Action). Otherwise IAAFT defers to Phase 7 and `null.rs` ships
   only `circular_shift_null_p`. The default plan-of-record is INCLUDE IAAFT in
   this plan; the test gate proves it ships.

4. **fdr.rs** — `bh_fdr(p_values: &[f64], alpha: f64) -> Vec<f64>` per
   Benjamini-Hochberg (1995) step-up. ~25 LOC pure function; hand-rolled (NOT
   the `adjustp` crate — 430 dl/mo, single-author, rejected per 05-RESEARCH §"Don't Hand-Roll").

5. **seed.rs** — `derive_job_seed(master_seed, scan_id_at_version, instruments,
   timeframe, window, param_hash) -> u64`. Reuses the existing Phase 2
   `blake3::Hasher` discipline. ~30 LOC including doc-comment.

Purpose: Decouples kernel mathematics from engine integration. Plans 05-03
(engine integration) and 05-04 (sweep runner) consume these functions as
opaque pure functions — they get a `f64`/`[f64;2]`/`Vec<f64>` back per call and
do not need to understand the internal resampling logic.

Output: Six new source files committed (`mod.rs` + five kernel sub-modules)
with their unit tests; `cargo test -p miner-core --lib scan::hygiene` exits 0;
no public API contract breaks; clippy clean.
</objective>
