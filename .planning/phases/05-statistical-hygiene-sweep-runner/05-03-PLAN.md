<objective>
Phase 5 Plan 03 wires the hygiene kernels (Plan 05-02) into the scan-execution
flow and gives every Phase 4 scan two new contracts:

A. **Effect-size emission (HYG-01).** Every `Finding::Result` MUST carry
   `effect.effect_size = Some(EffectSize { kind, value })` with the canonical
   `kind` string from the D5-03 table. The 22 Phase 4 scans each compute the
   appropriate scalar and assemble the `EffectSize` struct in their
   `Scan::run` body.

B. **Bootstrap + null opt-in (HYG-03, HYG-04, HYG-05).** Each scan overrides
   `supports_bootstrap()` and `supports_null_method(NullMethod)` per the
   per-scan matrix (Table below). The engine's `run_one_with_registry`
   invokes hygiene kernels AFTER the scan emits its base `Effect`, replaces
   `Effect.p_value` (if null requested) and/or populates `Effect.ci95`
   (if bootstrap requested), and populates `ResultFinding.repro` with the
   master + job seeds + method specs.

### Engine Integration Surface (D5-04, D5-05)

`ScanRequest` gains six additive `Option` fields (`#[serde(default)]`):
- `master_seed: Option<u64>` ŌĆö propagated from `[sweep].seed` or `--seed`.
- `job_seed: Option<u64>` ŌĆö derived via `hygiene::seed::derive_job_seed`.
- `bootstrap_method: Option<BootstrapMethod>` ŌĆö new enum mirroring `NullMethod`.
- `bootstrap_n: Option<u32>`.
- `null_method: Option<NullMethod>`.
- `null_n: Option<u32>`.

Single-shot `miner scan` (Plan 03-05) populates these from CLI flags (Plan 05-05 wires the flags). Sweep (Plan 05-04) populates them per-job from the resolved manifest.

`engine::run_one_with_registry` body extension (post-`Scan::run`):
1. If `req.bootstrap_method.is_some()` and `scan.supports_bootstrap() == true`: invoke `hygiene::bootstrap::stationary_bootstrap_ci` or `block_bootstrap_ci` per method, populate `effect.ci95`. The stat closure is constructed at the call site ŌĆö per-scan dispatch table.
2. If `req.null_method.is_some()` and `scan.supports_null_method(method) == true`: invoke `hygiene::null::circular_shift_null_p` or `iaaft_phase_scramble_null_p`, REPLACE `effect.p_value` with the empirical null p-value.
3. Populate `result.repro = Some(ReproEnvelope { master_seed, job_seed, bootstrap, null })` IFF either bootstrap or null ran.
4. Preflight rejection: if `req.bootstrap_method.is_some()` but `scan.supports_bootstrap() == false`, emit `Finding::ScanError` with `PreflightCode::HygieneNotSupported` and return (no scan body invocation, no `Result` emission). Same for `null_method`.
