<objective>
Phase 5 Plan 04 lands the sweep runner: TOML manifest deserialisation, cartesian
expansion, rayon-parallel job fanout with deterministic-order drain, end-of-sweep
BH-FDR aggregation, and the `Finding::SweepSummary` envelope emission.

Module decomposition under `crates/miner-core/src/sweep/`:

1. **`manifest.rs`** ‚Äî `serde::Deserialize`-derived typed struct tree for the
   TOML manifest schema documented in D5-01:
   - `SweepManifest { sweep: SweepConfig, hygiene: HygieneBlock, fdr: FdrConfig, jobs: Vec<JobBlock> }`
   - `SweepConfig { seed: Option<u64>, max_jobs: u64 (default 100_000) }`
   - `HygieneBlock { bootstrap: Option<String>, bootstrap_n: u32, null: Option<String>, null_n: u32 }`
   - `FdrConfig { family: String (default "scan_id"), alpha: f64 (default 0.05) }`
   - `JobBlock { scan, instruments: serde_json::Value, timeframes, windows, gap_policy, params: BTreeMap<String, serde_json::Value>, hygiene: Option<HygieneBlock> }`
   - `pub fn read_manifest(path: &Path) -> Result<SweepManifest, MinerError>` that maps `toml::de::Error` to `MinerError::Preflight(WireError::preflight(PreflightCode::InvalidParameter, ...))`.

2. **`job_graph.rs`** ‚Äî pure cartesian expansion:
   - `ResolvedJob { scan_id_at_version, instruments: Vec<InstrumentSpec>, timeframe: Timeframe, window: ClosedRangeUtc, gap_policy: GapPolicyKind, resolved_params: serde_json::Value, param_hash: String, job_seed: u64, bootstrap_method: Option<BootstrapMethod>, bootstrap_n: Option<u32>, null_method: Option<NullMethod>, null_n: Option<u32> }`
   - `pub fn expand(manifest: &SweepManifest, registry: &Registry) -> Result<Vec<ResolvedJob>, MinerError>` per 05-PATTERNS ¬ß"job_graph.rs" (lines 539‚Äì590).
   - Iteration order (D5-01): `[[jobs]]` block declaration order ‚Üí instruments (vector order) ‚Üí timeframes (vector order) ‚Üí windows (vector order) ‚Üí params (alphabetic key order). Within `params`, value arrays expand cartesian, also in alphabetic key order.
   - Arity validation per-`[[jobs]]`: registry-resolved `Scan::arity()` determines whether `instruments` is a flat `Vec<String>` (Single-arity) or a nested `Vec<Vec<String>>` (Pair-arity). Mismatches reject with `PreflightCode::InvalidParameter`.
   - Per-job `param_hash` computed via existing `engine::param_hash::param_hash(&resolved_params)`.
   - Per-job `job_seed` via `hygiene::seed::derive_job_seed`.

3. **`executor.rs`** ‚Äî `pub fn run_sweep` per 05-PATTERNS ¬ß"executor.rs" + RESEARCH Pattern 4:
   - Pre-execution: validate manifest, expand to `Vec<ResolvedJob>`, enforce `[sweep].max_jobs`.
   - Dry-run short-circuit: if `dry_run: true`, build a sample `DryRunFinding { planned_job_count: Some(jobs.len() as u64), ... }`, emit via sink, return.
   - Parallel execution: `rayon::par_iter().enumerate().map(|(idx, job)| { ... })` ‚Äî each worker calls `engine::run_one_with_registry` with a per-job `VecSink` capturing the job's findings. Workers honour the outer `cancel` flag.
   - Deterministic drain: collect `(idx, Vec<Finding>)` tuples, sort by idx, drain to the real sink. Capture `(scan_id_at_version, raw_p)` per `Result` envelope along the way for BH-FDR scoping.
   - SIGINT short-circuit: if cancel was set during execution, SKIP the `SweepSummary` emission and return ‚Äî CLI maps to exit 130. Already-streamed findings persist by virtue of the drain having already written them.
   - BH-FDR: group `(family_key, finding_index, raw_p)` by `family_key` per `[fdr].family`, call `hygiene::fdr::bh_fdr(&p_values, alpha)`, build `BTreeMap<String, FdrFamilySummary>`.
   - SweepSummary: emit `Finding::SweepSummary(SweepSummaryFinding { run_id, produced_at_utc, fdr_by_family, totals })`.

4. **`DryRunFinding` extension (`findings/mod.rs`)**: append `planned_job_count: Option<u64>` field with `#[serde(default)]`. Per RESEARCH Pattern 5 ‚Äî extending the existing variant (NOT introducing a new `SweepDryRun` variant). Single-shot `miner scan --dry-run` leaves the field `None`; sweep `--dry-run` populates it.

5. **Engine `run_sweep` wrapper (`engine/mod.rs`)** ‚Äî public entry point that bootstraps the registry and delegates to `sweep::executor::run_sweep_with_registry`. Mirrors `run_one` ‚Üí `run_one_with_registry` precedent.

6. **Integration tests** in `crates/miner-core/tests/`:
   - `sweep_smoke.rs`: 2 scans √ó 2 instruments √ó 1 tf √ó 1 window √ó 1 param sweep; assert envelope order RunStart ‚Üí 4 Results ‚Üí SweepSummary ‚Üí RunEnd; assert SweepSummary `fdr_by_family.len() == 2`.
   - `sweep_dry_run.rs`: `--dry-run` emits ONE `DryRunFinding` with `planned_job_count == 4`; no `Result`/`SweepSummary` envelopes follow.
   - `sweep_summary_emission.rs`: assert SweepSummary is strictly after the last Result and strictly before RunEnd; `method == "benjamini_hochberg"`; `per_finding` is stable index order.
   - `sweep_byte_identical_rerun.rs`: run the same sweep twice with `master_seed = 0xDEAD`; mask volatile fields; assert byte-equal JSONL.
   - `fdr_family_scoping.rs`: drive `[fdr].family = "scan_id" | "scan_family" | "all" | "none"`; assert per-variant the `fdr_by_family` key count + grouping.

Purpose: `OP-04` ships; the user can author a TOML manifest and `miner sweep` it from the CLI (Plan 05-05 adds the CLI plumbing). HYG-02 ships in production form (BH-FDR per family at end-of-sweep). HYG-05 byte-for-byte reproducibility extends to the sweep level.

Output: Sweep runner module + 5 integration tests committed; `cargo test --workspace` exits 0.
