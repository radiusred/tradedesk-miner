<objective>
Land the pure-function aggregator + the FX-major trading calendar. The aggregator transforms 1m `RawBar`s from any `Reader` impl into a column-oriented `BarFrame` at 15m / 1h / 1d resolution, with deterministic UTC bucketing and gap omission (never interpolation). The Calendar is the closed-form `is_open_at` predicate the aggregator + Plan 04's gap-detector both call.

Purpose: This is the central computation Phase 2 ships. Every later phase consumes `BarFrame`. Determinism is the contract — byte-identical Arrow IPC bytes across re-runs (CACHE-04) is the test Plan 05 will pin via snapshot.

       - Validate `params.range.start` is aligned to `params.tf.duration()` boundary (e.g., for 15m, `start.minute() % 15 == 0` AND `start.second() == 0` AND `start.nanosecond() == 0`). Return `MisalignedRange` if not.

       - Pull the 1m iterator from `reader.read_1m_bars(params.symbol, params.side, params.range)`. Map each `Err(RE)` → propagate as `Err(AggregateError::Reader(RE))` via `?`.
       - Walk the iterator in order. Maintain state `current_bucket_open: Option<DateTime<Utc>>` and accumulators `(open, high, low, close, tick_volume)`. For each `RawBar`:
         - Compute the bucket open for this minute: `bucket_open = align_down(bar.ts_open_utc, params.tf)`.

         **`align_down` — single strategy across all three timeframes (revision pass 1 — replaces the earlier mixed-strategy guidance):** truncation is component-wise via chrono's `.with_*` setters because the aggregator works in UTC throughout — chrono UTC has no DST math, so component truncation is byte-stable across all timeframes. The helper:

         ```
         fn align_down(ts: DateTime<Utc>, tf: Timeframe) -> DateTime<Utc>
         ```

         For every timeframe, start by zeroing the sub-minute components: `ts.with_second(0).unwrap().with_nanosecond(0).unwrap()` — call this `t0`. Then:
         - `Tf15m`: `t0.with_minute((t0.minute() / 15) * 15).unwrap()` (floor-to-15-minute boundary).
         - `Tf1h`: `t0.with_minute(0).unwrap()`.
         - `Tf1d`: `t0.with_minute(0).unwrap().with_hour(0).unwrap()`.

         Do NOT use `date_naive().and_hms_opt(0,0,0).unwrap().and_utc()` for Tf1d — keep the component-wise form for byte-stability uniformity across all three timeframes (single code path, single test surface). The aggregator is UTC-only so there is no DST concern, but the component-wise form removes the need to justify a per-timeframe branch in code review.

         - If `current_bucket_open.is_none() || bucket_open != current_bucket_open.unwrap()`: emit the previous bucket (if any) by pushing all 7 accumulator values onto the BarFrame's Vec columns; start a new bucket with `current_bucket_open = Some(bucket_open)` and accumulators initialised to the current bar's values.
         - Otherwise (same bucket): update `high = high.max(bar.high)`, `low = low.min(bar.low)`, `close = bar.close`, `tick_volume = tick_volume + bar.tick_volume`. **Sequential f64 sum in iteration order — this is the determinism guarantee for non-associative f64 addition.** `open` stays as the bucket's first bar's open.
       - After the loop, emit the final bucket if any.
       - Return `Ok(frame)`.
