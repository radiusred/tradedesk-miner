Land the derived-bar cache: Arrow IPC files per `(source_id, symbol, side, timeframe)` quartet + a sidecar JSON containing per-day blake3 fingerprints. The cache reads-mostly; writes are crash-safe via the tempfile-rename pattern; invalidation is two-axis (full-rebuild on aggregator_version / arrow_schema_version mismatch; day-splice on per-day fingerprint mismatch).

    2. `impl BarCache { pub fn get_or_build<R: Reader>(&self, reader: &R, params: AggParams) -> Result<BarFrame, CacheError> { ... } }`:

       Algorithm:
       a. Compute `arrow_path = cache::arrow_path(&self.cache_root, reader.source_id(), params.symbol, params.side, params.tf)`. Compute `sidecar_path = cache::sidecar_path(&arrow_path)`.
       b. Read existing sidecar: `let existing_sidecar = fingerprints::read_sidecar(&sidecar_path)?;`. None if not present.
       c. Decide full_rebuild:
          - If existing_sidecar is None → full_rebuild = true.
          - Else if `existing_sidecar.aggregator_version != AGGREGATOR_VERSION || existing_sidecar.arrow_schema_version != ARROW_SCHEMA_VERSION` → full_rebuild = true. Tracing: `tracing::info!(?old_v = ..., ?new_v = ..., "cache invalidated by version bump, rebuilding")`.
          - Else full_rebuild = false.
       d. Walk source days in range: `let current_days = reader.enumerate_days(params.symbol, params.side, params.range).map_err(|e| CacheError::Aggregate(e.to_string()))?;`. For each, call `reader.fingerprint_day(symbol, side, date)` (NOT rayon-parallelised in v1 — keep sequential for determinism and simplicity; RESEARCH suggests rayon but flags it as an optimisation). Build `current_fingerprints: BTreeMap<NaiveDate, Blake3Hex>`.
       e. If full_rebuild → stale_days = all current_days. Else → `stale_days = fingerprints::diff_days(&existing_sidecar.unwrap().per_day_fingerprint, &current_fingerprints)`.
       f. If `stale_days.is_empty()` → cache hit. Read existing Arrow file via helper `read_arrow_file(&arrow_path)` (Task 2 helper, see below) → return BarFrame.
       g. Else → need to (re)build:
          - If full_rebuild OR `!arrow_path.exists()`: aggregate the entire range. Call `aggregate(reader, AggParams { symbol, side, tf, range })`. Convert BarFrame → RecordBatch (helper `bar_frame_to_record_batch(&frame, &schema) -> Result<RecordBatch>`). Write via `write_arrow_atomic`.
          - Else (day-splice): read existing Arrow → BarFrame; drop rows whose `ts_open_utc.date_naive()` is in stale_days; for each stale_day, aggregate JUST that day; concatenate surviving + new rows (sorted by ts_open_utc — use Vec::sort_by_key); write via write_arrow_atomic.
       h. Update sidecar:
          - `let new_sidecar = FingerprintSidecar { aggregator_version: AGGREGATOR_VERSION.into(), arrow_schema_version: ARROW_SCHEMA_VERSION.into(), source_id: reader.source_id().into(), symbol: params.symbol.into(), side: params.side, timeframe: params.tf, per_day_fingerprint: current_fingerprints.into_iter().map(|(d, fp)| (d, fp.as_str().to_string())).collect() };`
          - `fingerprints::write_sidecar_atomic(&sidecar_path, &new_sidecar)?;`
       i. Read back the just-written Arrow file → return BarFrame. (Alternative: keep the in-memory BarFrame from the aggregation. The read-back proves the round-trip works but adds I/O. **Use the in-memory BarFrame for performance**; the round-trip is covered by the cache-hit path in test 1.)
