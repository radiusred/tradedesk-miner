    2. `impl GapDetector { pub fn detect<R: Reader>(reader: &R, symbol: &str, side: Side, range: ClosedRangeUtc) -> Result<GapManifest, R::Error> { ... } }` — the core function:

       Algorithm:
       a. `let calendar = reader.trading_calendar();`
       b. `let mut gaps: Vec<GapSpan> = Vec::new();`
       c. Enumerate calendar dates in range (every NaiveDate from `range.start.date_naive()` up to and including `range.end.date_naive()` if end is at midnight, or just `range.end.date_naive()` if end is past midnight; document the half-open convention).
       d. For each date, compute the day's calendar minutes (0..1440). Compute `open_minutes`: the subset of those minutes whose UTC timestamp `calendar.is_open_at(...)` returns true. If `open_minutes.is_empty()` → entire day closed → no gap. Skip.
       e. Else: call `reader.fingerprint_day(symbol, side, date)`. Match:
          - `Ok(None)` → file absent → push `GapSpan { start_utc: date.and_hms(0,0,0).and_utc(), end_utc: date.succ_opt().unwrap().and_hms(0,0,0).and_utc(), reason: GapReason::MissingSourceFile { date } }`. **Per D2-17: do NOT also push intra-day gaps for this date — they'd be redundant.**
          - `Ok(Some(_))` → file present → call `reader.read_1m_bars(symbol, side, day_range)` to enumerate the bars within the day. Detect "zero-byte/corrupt" indirectly: if the iterator's first result is `Err(_)` for the file, push `GapSpan { ..., reason: GapReason::CorruptSourceFile { date, detail: error_string } }`. Actually — the cleaner separation is: `fingerprint_day` returns `Ok(Some(hex))` for a present-and-readable file; if the file is zero-byte the Reader's path-level enumeration should still surface it via `read_1m_bars` returning `Err`. **Reality:** `reader.fingerprint_day` reads the file bytes (Plan 01 Task 4) and SUCCEEDS on a zero-byte file (blake3 of empty is a valid hex). So the "file present but corrupt" detection happens at `read_1m_bars` time. Two options:
            - Option A (simpler): Treat `read_1m_bars` error per day as CorruptSourceFile. Push the GapSpan and continue.
            - Option B (more rigorous): Reader's `fingerprint_day` returns a special sentinel (e.g., `Blake3Hex::ZERO_BYTE_SENTINEL`) for zero-byte files. Cleaner separation but adds API surface.
            Use Option A for v1. Document the trade-off in a `// TODO(Phase 7): consider explicit zero-byte sentinel` comment.
          - `Err(e)` → propagate as `Err(e)` — these are I/O errors at the boundary, not gaps.
          - After processing the day's bars, compute per-minute presence: build a `BTreeSet<DateTime<Utc>>` of present-bar `ts_open_utc`. For each `open_minute` in the day, check presence. Coalesce contiguous-missing-minutes into single GapSpans: `[12:00, 12:01) ∪ [12:01, 12:02) → [12:00, 12:02)`. Push as `GapReason::IntraDayGap { affected_minutes: contiguous_count }`. **Per D2-17 v1 recommendation: do NOT merge adjacent intra-day gaps across non-contiguous minutes. Keep contiguous single-minute runs as one entry.**
          - Actually re-read RESEARCH §"Gap Manifest Data Model" line 865: "**Recommendation for v1: do NOT merge adjacent intra-day gaps.** Keep them as individual one-minute entries." → **for v1, emit one IntraDayGap entry per missing minute** (no coalescing). `affected_minutes = 1` for each. Simpler is correct.

The gap detector's intra-day-gap logic generates per-minute gap entries; the aggregator (Plan 02) computes whether the bucket is >50% missing.
