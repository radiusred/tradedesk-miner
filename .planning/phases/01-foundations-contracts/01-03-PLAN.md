Implement the locked Finding envelope as Rust types: the tagged enum with five variants, all locked common fields, RawArray composition, RunStart/RunEnd framing payloads, RunId ULID newtype, Base64Bytes newtype (using the verified spike pattern), and the error_code vocabulary enums. This is the centrepiece plan of Phase 1: every later phase consumes these types. Threat T-01-02 (schema injection / drift) is mitigated by ensuring the Rust types ARE the source of truth and they derive `JsonSchema` so Plan 06's xtask can regenerate the schema from them. Threat T-01-04 (code revision tampering) flows through to the envelope via `code_revision: String` field populated from `miner_core::CODE_REVISION`.

**Output:** All five envelope variants + supporting types compile, derive `Serialize + Deserialize + JsonSchema`, and `cargo test -p miner-core findings::` round-trips one instance of each variant. The lib.rs `pub use` block exports every name Plan 05 and Plan 06 consume.
</objective>
